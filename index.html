CSS

  • Enter your CSS notes here
  • Cascading Style Sheets (CSS) is a programming language used developers to define how webpage content is presented to users
  • CSS defines the colors, aesthetics, visual layout, and organization of a webpage
  • CSS is used to define style rules for specific elements of an HTML page and format them using a wide range of style options
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet
  • Inline CSS
    • An inline CSS is used to apply a unique style to a single HTML element
    • An inline CSS uses the style attribute of an HTML element
    • Useful for quick and permanent changes but less flexible than external and internal style sheets
    • Inline style must be edited individually whenever a design change is made
  • Internal CSS style sheet
    • An internal CSS is used to define a style for a single HTML page.
    • It embeds an entire stylesheet directly in the HTML file using a "<style>" element in the "<head>" element
    • It contains CSS rules for the webpage in the head section of the HTML file
    • The rules only apply to that page, and you can configure CSS classes and IDs, which are used to identify elements, to style multiple elements in the page code
    • A single change to the CSS rule will apply to all tagged page elements
  • External CSS
    • This is the most common and prefered way to style a webpage
    • An external style allows developers to keep all their CSS rules in a separate file, which makes design changes easier
    • When using an external style sheet, we link the file using a "<link>" element placed in the HTML's "<head>"
    • The external style sheet can be written in any text editor. The file must not contain any HTML code, and must be saved with a .css extension.
    • Seperating code into different files based on use is known as separation of concerns. It is based on the idea that each section of code should have its own responsibility

Git

  • Enter your Git notes here
  • Git is a version control system that allows you to manage and keep track of each version of a project
    • Every time a version of our work is saved using version control, a unique identifier called an SHA, or hash, allows you to keep a record of the specific changes committed
  • We access Git using the command line
    • Unlike bash commands, Git commands always start with “git” and are used to save and track versions of our code
  • Git commands can be used to initiate (create) a repository, add code to the repository, and view different versions of our code, which are each stored in a unique commit
    • You can find a list of the most important and commonly used Git commands here: Git Command Cheat Sheet
  • While git can track changes locally on our computer, collaborating with others requires a tool called GitHub
    • GitHub is a cloud-based hosting service that allows us to keep track of our projects inside repositories that are stored remotely.
  • Developers use repositories to store the following:
    • Files that contain code
    • Files that contain other information about the code, like a README file
    • The version history for all of the files in the repository
  • When making changes locally, it is important to keep the local branch in sync with the remote repository/li>
  • To keep the two in sync, we use the GitHub workflow
  • A branch is a copy or version of the code that can be worked on independently in isolation so development changes do not disrupt other team members who are also working on the code and testing their solutions
  • Every GitHub repository starts with a default branch called main
    • From the main branch, we can create new branches called feature branches where changes can be made without disrupting the main branch
    • In a feature branch, all changes made should include a commit message that is descriptive of what changed in the code
  • Changes made via feature branches are only integrated after merging into the main branch
    • A pull request, or PR, is used to merge into the main branch
  • You can read more about the different stages of GitHub flow and branching here: GitHub Docs on GitHub flow

JavaScript

  • Enter your JavaScript notes here
  • JavaScript (JS) is a high-level dynamic programming language that allows users to interact with websites
  • Using JavaScript, you can power your websites with effects, interactivity, data communication, and much more
  • JavaScript takes a static HTML/CSS webpage and makes it interactive
  • Variables are containers that allow us to store data in our code
    • To declare a variable in JavaScript, we us the var key word, followed by the variable name
    • The variable name (identifiers) should be unique as it references the information stored in the variable
    • After declaring and naming a variable, the assignment operator = is used to assign the specific data that we intend to store in our variable
  • A JavaScript variable can hold many different types of data that can be categorized into two main groups: primitive types and object types
  • Primitive Types:
    • String - Represents sequences of characters ("How are you?")
    • Number - Represents numeric values. These values can either be a integerNumber (1, 10 100) or a floatingPointNumber (3.14, 0.50. 9.99)
    • Boolean - Represents a logical value, either true or false
    • Undefined - Represents an unassigned or undefined value
    • Null - Represents an empty value, similar to undefined, but is explicitly assigned to indicate the intentional absence of any object value
  • Object Types:
    • Object - Represents a collection of key-value pairs, where an identifier or key is paired with its corresponding value
    • Array - Represents an ordered list of values
    • Function - Represents a reusable block of code
  • You can read more about JavaScript basics and data types below:
    • MDN Web Docs on JavaScript basics
    • MDN Web Docs on JavaScript data types
